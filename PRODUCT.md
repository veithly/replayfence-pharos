# Product

## Register

product

## Users

ReplayFence serves AI agent developers, Pharos hackathon judges, and protocol reviewers. They need to install a reusable skill, run one risky action once, replay it, and inspect a proof capsule without building or trusting a bespoke agent.

## Product Purpose

ReplayFence turns duplicate agent retries into typed, proof-carrying rejections. The P0 product is an OpenClaw-installable skill plus a Pharos once-latch backend and a small visual workbench for capsule inspection.

## Brand Personality

Precise, calm, testbench-like. The product should feel like a trustworthy safety instrument, not a crypto marketing page.

## Anti-references

Avoid generic Web3 dashboards, wallet-first token apps, AI chat copilots, purple crypto gradients, over-decorated SaaS cards, and fake terminal output that is not backed by real commands.

## Design Principles

1. Skill first: the installable OpenClaw skill is the product, the web UI is evidence.
2. Show the state machine: ready, consumed, replay rejected, capsule ready.
3. Be honest about proof: local demo, live tx, proof fetched, and proof verified are distinct states.
4. Keep Pharos visible where it changes trust, not as a badge.
5. Make every claim exportable: transcript, capsule, tx link, verifier report, or test result.

## Accessibility & Inclusion

Target WCAG AA. Status changes must use text and aria-live, not color alone. Motion is short and state-driven with reduced-motion fallback. Long hashes need copy buttons and readable labels.
