## Synthesis For This Hackathon

- Project families to generate:
  - Proof-carrying paid verifier skill.
  - Conditional action authorizer.
  - Judge/referee skill with verdict receipt.
  - Registrable skill with metadata, signature, price, and callable endpoint.
  - Human-backed exception handler for privileged actions.
  - SPN-aware execution router.
- Mature-product loops to adapt:
  - x402 paid endpoint.
  - MCP-like tool schema.
  - Sigstore-style attestation.
  - Request-style receipt object.
  - Verifiable compute / proof report.
- Host surfaces to prefer:
  - API workbench.
  - Agent call console.
  - Proof receipt detail.
  - Registry/history view.
  - Explorer/proof verification panel.
- Proof artifacts to require:
  - `PAYMENT-RESPONSE` or simulated payment receipt when no live ERC-20 is available.
  - Pharos Atlantic tx hash or explorer link.
  - `eth_getProof` response and independent verifier report.
  - Signed skill manifest or result digest.
  - Agent-call transcript with input, policy, output, and receipt IDs.
- Clone traps to forbid:
  - Wallet chat assistant.
  - Chain dashboard with no first-minute consequence.
  - RAG or summary app with no paid call and no durable artifact.
  - Marketplace landing page with no call/settlement/verifier loop.
  - Monolithic multi-agent app that hides the reusable skill.
  - Proof feature that only relays RPC output.
- Evidence gaps:
  - DoraHacks detail page was WAF-protected in command-line fetches; GPT Pro cites event page/snippet evidence and Pharos public Agent Carnival page.
  - Live x402 facilitator/token availability must be validated during build; fallback must be explicit if unavailable.
  - Live deployment will need a public host and at least one seeded fresh-user path.
